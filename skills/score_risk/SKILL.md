# 分数位次风险分析规范

## 适用场景

用户提供或询问分数、位次、历年录取线、目标专业录取可能性、冲稳保判断时使用本 Skill。

## 回复原则

- 优先依据知识库、记忆上下文和 MCP 工具结果；没有依据时说明信息边界。
- 位次优先于分数；同一分数在不同省份、年份、科类或选科下不可直接比较。
- 必须优先依据 MCP 录取风险评估结果和知识库历年数据。
- 如果 MCP 状态不是 ok，只能追问缺失信息或说明暂无法自动判断，不得生成历年数据和风险结论。
- 风险判断只能用“冲、稳、保、风险较高、需要谨慎、建议结合位次确认”等倾向性表达。
- 不得承诺录取概率的精确百分比，除非工具或官方数据明确给出且注明来源。

## 处理流程

1. 检查是否具备省份、科类或选科、分数、位次、目标专业、批次。
2. 若缺少省份、科类或位次，优先追问；不要直接给冲稳保结论。
3. 若 MCP 返回 need_more_info，复述缺失项并解释为什么需要这些信息。
4. 若 MCP 返回 ok，引用年份、最低分、最低位次、计划数和工具给出的风险判断。
5. 给出建议时区分学校层面和专业层面，提醒填报梯度和服从调剂风险。

## 必须追问的信息

- 省份。
- 科类或选科组合，例如物理类、历史类、理科、文科。
- 高考分数和全省位次，位次缺失时提醒查询一分一段表。
- 目标专业或专业组，若只问学校层面则说明只能先做学校层面判断。
- 批次和是否接受调剂。

## 升级条件

- 用户需要最终录取结论或官方保证。
- 数据库缺少该省份、科类、年份或目标专业历史数据。
- 用户涉及强基、专项、艺术体育、民族预科、中外合作等特殊录取类型。
- 用户临近志愿截止且需要确认当年招生计划或专业组设置。

## 禁止事项

- 禁止编造最低分、最低位次、计划数、平均分或录取概率。
- 禁止使用 XXX、YYY、假设数据占位。
- 禁止在只有分数、没有省份和位次时做确定判断。
- 禁止把往年最低位次当成今年录取保证。
